  <name>Task 1: Deploy Prometheus + Grafana via Helm</name>
  <files>infra/k8s/helm-values/prometheus-grafana.yaml, tasks.py</files>
  <action>
  Add kube-prometheus-stack Helm chart to Kind cluster:
  - Create infra/k8s/helm-values/prometheus-grafana.yaml with:
    - Grafana admin password from env var (default: admin for local dev)
    - Grafana service type NodePort on port 3000
    - Pre-provisioned dashboard for K8s pod metrics (JSON dashboard configmap)
    - Prometheus retention: 24h (local dev, save disk)
  - Add invoke task: deploy_observability (helm install kube-prometheus-stack)
  - helm install prometheus prometheus-community/kube-prometheus-stack -n null-realm -f infra/k8s/helm-values/prometheus-grafana.yaml

  Do NOT use grafana.ini for config — use Helm values only.
  </action>

  <done>Grafana responds on localhost:3000, Prometheus scraping K8s metrics</done>
</task>

<task type="auto">
  <name>Task 2: Deploy Jaeger for distributed tracing</name>
  <files>infra/k8s/system/jaeger/deployment.yaml, infra/k8s/system/jaeger/service.yaml</files>
  <action>
  Deploy Jaeger all-in-one (fine for dev — single container with in-memory storage):
  - Deployment: jaegertracing/all-in-one:latest image
  - Ports: 16686 (UI), 4317 (OTLP gRPC), 4318 (OTLP HTTP)
  - Service: NodePort exposing 16686 for UI access
  - Resource limits: 512Mi memory, 500m CPU (local dev)

  Do NOT use Jaeger operator — overkill for dev. Use simple deployment.
  </action>
  <verify>kubectl get pods -n null-realm | grep jaeger && curl -s localhost:16686/</verify>
  <done>Jaeger UI accessible at localhost:16686</done>
</task>

<task type="auto">
  <name>Task 3: Deploy Langfuse (self-hosted) for LLM tracing</name>
  <files>infra/k8s/system/langfuse/deployment.yaml, infra/k8s/system/langfuse/service.yaml</files>
  <action>
  Deploy Langfuse self-hosted on Kind:
  - Langfuse needs PostgreSQL — deploy a small PostgreSQL StatefulSet (separate from app DB) OR use a single shared PG instance
  - Recommendation: single PostgreSQL StatefulSet in null-realm namespace, shared by Langfuse + app later
  - Create infra/k8s/system/postgres/statefulset.yaml: PostgreSQL 16 with pgvector extension, 1Gi PVC
  - Create Langfuse deployment: langfuse/langfuse:latest image
  - Env vars: DATABASE_URL pointing to local PG, NEXTAUTH_SECRET, SALT, NEXTAUTH_URL=http://localhost:3001
  - Service: NodePort on 3001

  Do NOT enable Langfuse cloud features — this is self-hosted only.
  </action>
  <verify>kubectl get pods -n null-realm | grep -E "(langfuse|postgres)" && curl -s localhost:3001/</verify>
  <done>Langfuse UI accessible at localhost:3001, connected to PostgreSQL</done>
</task>
