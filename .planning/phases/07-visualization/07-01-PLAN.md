# Plan: Knowledge Graph Visualization — "Null Realm Universe"

## Context

We have a Neo4j knowledge graph with ~10,751 code symbols across 11 repos + service topology (Service, Endpoint, Topic, InfraService nodes). The data is queryable via MCP tools (code_search, graph_query, service_topology), but there's no visual way to explore it.

**The vision**: An interactive graph visualization where you can:
- See the full Scality service topology at a glance
- Zoom into a service → see its code symbols, endpoints, configs
- See cross-repo XREF edges connecting shared dependencies
- Navigate from "bucket notifications" → backbeat notification extension → Kafka topics → cloudserver routing
- Use it as an architectural thinking tool — rubber duck different designs

Think: Obsidian graph view meets a universe visualization, purpose-built for the Scality S3 platform.

## Research Findings

| Tool | Nodes | Style | Integration | Effort |
|------|-------|-------|------------|--------|
| Grafana Node Graph | ~100 | 2D directed graph | Native Grafana | 1 day |
| streamlit-agraph | ~5K | 2D force-directed | Drop into existing Streamlit | 2-3 days |
| pyvis | ~5K | 2D force-directed (HTML export) | Standalone | 2 days |
| react-force-graph-3d | ~50K | 3D WebGL universe | New React app | 1-2 weeks |
| cosmograph | ~100K+ | GPU-accelerated WebGL | New React app | 1-2 weeks |
| Neo4j Bloom | ~100K+ | Professional graph explorer | Neo4j Enterprise ($$) | 1 day |

**At 10K nodes + 50K edges**, streamlit-agraph and pyvis struggle. Cosmograph and react-force-graph-3d handle it comfortably with GPU acceleration.

## Recommended Approach: Three Tiers

### Tier 1: Grafana Service Topology Dashboard (Phase 07-01)

**What**: A Grafana dashboard that visualizes service-level topology from Neo4j.
**Why now**: Quick win, gives immediate visibility, uses existing Grafana.
**How**: 
- Neo4j data source plugin for Grafana (community plugin)
- OR: FastAPI endpoint that returns graph data → Grafana Node Graph panel
- Shows: Service nodes (colored by type), DEPENDS_ON/HTTP_CALLS/USES_CLIENT edges
- Overlay: chunk counts as node size, indexing status as color
- Panels: service topology graph, dependency matrix table, repo stats

**Effort**: 1-2 days

### Tier 2: Streamlit Knowledge Graph Explorer (Phase 07-02)

**What**: Add a "Graph Explorer" tab to the existing embedding-viz Streamlit app.
**Why**: Extends existing infrastructure, interactive, good for focused exploration.
**How**:
- `streamlit-agraph` component for force-directed graph rendering
- Neo4j Python driver queries
- Features:
  - **Search box** → find symbol/service → show its neighborhood graph
  - **Service view** → select a service → see all its symbols, endpoints, deps
  - **Cross-repo view** → select two repos → show XREF edges between them
  - **Filters** → by repo, symbol type, relationship type, language
  - **Node detail panel** → click a node → see code snippet, file path, connections
- Works well when filtered to <5K nodes (one repo, one service, etc.)

**Effort**: 3-5 days

### Tier 3: "Null Realm Universe" — 3D Graph Explorer (Phase 07-03)

**What**: A standalone React app with GPU-accelerated 3D graph visualization.
**Why**: The "wow" — architecturally beautiful, handles full 10K+ node graph.
**How**:
- **cosmograph** for rendering (GPU-accelerated WebGL, handles 100K+ nodes)
- React + TypeScript frontend
- FastAPI backend (or direct Neo4j browser driver)
- Features:
  - **Universe view**: Every symbol is a star. Repos are galaxies (color-coded clusters). Services are constellations. XREF edges are bridges between galaxies.
  - **Semantic zoom**: Zoom out → see service topology. Zoom in → see functions/classes.
  - **Search + highlight**: Type "MetadataWrapper" → pulses across all repos where it's used.
  - **Path tracing**: Click two nodes → animate the shortest path between them (across repos).
  - **Time dimension**: Color by last_indexed_at → see what's fresh vs stale.
  - **Filter panels**: Toggle relationship types, repos, languages on/off.
- Deploy as new K8s pod on GKE: `universe.34.53.165.155.nip.io`

**Architecture**:
```
universe.nip.io (React + cosmograph)
  ├── /api/graph     ← FastAPI endpoint returning graph data
  │   └── Neo4j Cypher queries
  ├── /api/search    ← Symbol search endpoint  
  │   └── pgvector semantic search
  └── /api/path      ← Shortest path endpoint
      └── Neo4j shortestPath
```

**Effort**: 1-2 weeks

## Data Pipeline for Visualization

All three tiers query the same Neo4j graph. The key queries:

```cypher
-- Service topology (Tier 1 Grafana)
MATCH (a:Service)-[r]->(b)
RETURN a, r, b

-- Symbol neighborhood (Tier 2 Streamlit)
MATCH path = (a:Symbol {name: $name})-[:RELATES|XREF*1..2]-(b)
RETURN path

-- Full universe (Tier 3 cosmograph)
MATCH (n)
OPTIONAL MATCH (n)-[r]->(m)
RETURN n, r, m
LIMIT 50000

-- Cross-repo bridges (highlight view)
MATCH (a:Symbol)-[r:XREF]->(b:Symbol)
WHERE a.repo <> b.repo
RETURN a, r, b
```
