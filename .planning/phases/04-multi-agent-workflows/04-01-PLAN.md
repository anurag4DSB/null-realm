## Reality Notes (added post-execution)

What actually happened when this plan was executed:
- Removed S3 artifact repo from Helm values -- not needed on Kind, simplified the deployment
- Worker bootstrap didn't have init_tracing() initially -- agent pod traces were invisible in Langfuse/Jaeger until this was added
- Argo v4 metrics endpoint uses HTTPS (not HTTP) -- ServiceMonitor needed `tlsConfig.insecureSkipVerify: true` to scrape metrics
- Argo server needed `--managed-namespace=null-realm-agents` flag for the UI to show workflows in the correct namespace
- Argo v4 has no dark mode -- only `navColor` customization is available
- Pre-built Grafana dashboards used v3 metric names (e.g., `argo_workflows_count`) -- Argo v4 renamed these (e.g., `argo_workflows_gauge`). Built a custom v4 dashboard from scratch
- Also deployed to GKE with Argo UI accessible at argo.34.53.165.155.nip.io (not just Kind as the plan described)

See SUMMARY.md for full details.
