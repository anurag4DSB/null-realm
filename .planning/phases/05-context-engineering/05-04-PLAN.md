# Plan: Multi-Language Indexing (JS/TS/Go) via tree-sitter

## Context

The indexer (`nullrealm/context/indexer.py`) currently only parses Python files using the `ast` module. Scality's repos (cloudserver, Arsenal, etc.) are primarily JavaScript/TypeScript and Go. When we indexed cloudserver, only 7 Python utility files (monitoring dashboards, test scripts) were found ÔÇö the core S3 logic in `.js` files was ignored entirely.

**Goal**: Add JS/TS/Go parsing via tree-sitter so `index_repo` extracts functions, classes/structs, imports, and relationships from JavaScript, TypeScript, and Go files.

## Architecture Decision: tree-sitter

**Why tree-sitter** (not regex, babel subprocess, or esprima):
- Universal AST parser ÔÇö same API for JS, TS, Go, Rust, Java, etc.
- Python bindings (`tree-sitter` package) ÔÇö runs in-process, no subprocess overhead
- Used by GitHub (code navigation), Neovim, Helix ÔÇö battle-tested
- Language grammars are separate packages (`tree-sitter-javascript`, `tree-sitter-typescript`)
- Adding a new language = install grammar + write a ~100-line parser module

**Parser architecture**: Plugin-style dispatching by file extension:
```
indexer.py:
  .py  Ôćĺ parsers/python_parser.py  (existing ast module, unchanged)
  .js  Ôćĺ parsers/js_parser.py      (new, tree-sitter)
  .ts  Ôćĺ parsers/ts_parser.py      (new, tree-sitter)
  .go  Ôćĺ parsers/go_parser.py      (new, tree-sitter)
```

## What JS/TS Parsing Extracts

| Construct | Symbol Type | Example |
|-----------|-------------|---------|
| `function foo()` | function | `function validateBucketPolicy(...)` |
| `const foo = () =>` | function | `const parseRequest = (req) => ...` |
| `const foo = function()` | function | `const handler = function(req, res)` |
| `class Foo` | class | `class BucketPolicy extends ...` |
| `class Foo { method() }` | function | `BucketPolicy.validate` |
| `module.exports` | module | Module-level export |
| `import x from 'y'` | ÔÇö | IMPORTS relationship |
| `require('y')` | ÔÇö | IMPORTS relationship |
| `extends Bar` | ÔÇö | INHERITS relationship |
| `foo.bar()` calls | ÔÇö | CALLS relationship |

### Task 2: Create parser module structure

**Files**: `nullrealm/context/parsers/__init__.py`, `nullrealm/context/parsers/python_parser.py`

Create `nullrealm/context/parsers/` package. Move `parse_python_file()` from `indexer.py` into `parsers/python_parser.py` ÔÇö keep the exact same logic, just relocate. The `__init__.py` exports a dispatcher:

```python
# parsers/__init__.py
EXTENSION_MAP = {
    ".py": "nullrealm.context.parsers.python_parser",
    ".js": "nullrealm.context.parsers.js_parser",
    ".jsx": "nullrealm.context.parsers.js_parser",
    ".ts": "nullrealm.context.parsers.ts_parser",
    ".tsx": "nullrealm.context.parsers.ts_parser",
    ".go": "nullrealm.context.parsers.go_parser",
}

def parse_file(file_path: str) -> tuple[list[CodeChunk], list[CodeRelationship]]:
    ext = Path(file_path).suffix
    module_name = EXTENSION_MAP.get(ext)
    if not module_name:
        return [], []
    module = importlib.import_module(module_name)
    return module.parse_file(file_path)
```

### Task 3: Create JS parser with tree-sitter

**Files**: `nullrealm/context/parsers/js_parser.py`

Parse `.js` and `.jsx` files using `tree-sitter-javascript`:

1. Initialize tree-sitter JS language (cached, one-time)
2. Parse source into AST
3. Walk tree to extract:
   - **Functions**: `function_declaration`, `arrow_function`, `function` (assigned to variable)
   - **Classes**: `class_declaration` with methods (`method_definition`)
   - **Imports**: `import_statement`, `call_expression` where callee is `require`
   - **Exports**: `export_statement` (for module-level context)
   - **Calls**: `call_expression` nodes (function name extraction)
   - **Inheritance**: `class_heritage` / `extends` clause
4. Build `CodeChunk` and `CodeRelationship` objects (same dataclasses as Python parser)
5. Set `language="javascript"` on chunks

Key tree-sitter query patterns:
```
(function_declaration name: (identifier) @name) @func
(class_declaration name: (identifier) @name) @class
(import_statement source: (string) @source)
(call_expression function: (identifier) @callee (#eq? @callee "require"))
```

### Task 4: Create TS parser with tree-sitter

**Files**: `nullrealm/context/parsers/ts_parser.py`

Similar to JS parser but uses `tree-sitter-typescript` grammar. Additional constructs:
- `interface` declarations Ôćĺ class-like chunks
- Type annotations (stripped from chunk text for cleaner embedding)
- `enum` declarations Ôćĺ class-like chunks
- `namespace` / `module` declarations

Can share ~80% of logic with JS parser via a common `_parse_ecmascript()` helper.

### Task 5: Create Go parser with tree-sitter

**Files**: `nullrealm/context/parsers/go_parser.py`

Parse `.go` files using `tree-sitter-go`:

1. Initialize tree-sitter Go language (cached, one-time)
2. Parse source into AST
3. Walk tree to extract:
   - **Functions**: `function_declaration` Ôćĺ `func Foo(...)` 
   - **Methods**: `method_declaration` Ôćĺ `func (r *Receiver) Foo(...)` ÔÇö stored as `ReceiverType.Foo`
   - **Structs**: `type_declaration` with `struct_type` Ôćĺ treated as class-like chunks
   - **Interfaces**: `type_declaration` with `interface_type` Ôćĺ class-like chunks
   - **Imports**: `import_declaration` Ôćĺ IMPORTS relationships
   - **Calls**: `call_expression` Ôćĺ CALLS relationships
   - **Embeds**: struct embedding (anonymous fields) Ôćĺ INHERITS-like relationship
4. Build `CodeChunk` (language="go") and `CodeRelationship` objects
5. Handle Go-specific patterns:
   - Package-level `init()` functions
   - Exported vs unexported (capitalization) ÔÇö store both but note in metadata
   - Multi-return functions, error handling patterns

| Go Construct | Symbol Type | Example |
|-------------|-------------|---------|
| `func Foo()` | function | `func HandleS3Request(...)` |
| `func (s *Server) Foo()` | function | `Server.HandleS3Request` |
| `type Foo struct` | class | `type BucketPolicy struct` |
| `type Foo interface` | class | `type StorageBackend interface` |
| `import "pkg"` | ÔÇö | IMPORTS relationship |
| Struct embedding | ÔÇö | INHERITS relationship |

### Task 6: Update indexer to use multi-language parsing

**Files**: `nullrealm/context/indexer.py`

Update `index_repo()`:
1. Instead of `repo.rglob("*.py")`, collect files for ALL supported extensions:
   ```python
   SUPPORTED_EXTENSIONS = {".py", ".js", ".jsx", ".ts", ".tsx", ".go"}
   source_files = [p for p in repo.rglob("*") 
                   if p.suffix in SUPPORTED_EXTENSIONS
                   and not any(part in SKIP_DIRS for part in p.parts)]
   ```
2. Add more dirs to SKIP_DIRS: `"dist"`, `"build"`, `".next"`, `"coverage"`, `"vendor"`
3. Call `parsers.parse_file()` instead of `parse_python_file()`
4. Keep `parse_python_file()` in indexer.py as a deprecated alias (import from parsers)
5. Log language breakdown: "Found 150 JS, 30 TS, 7 Python, 45 Go files"
