  <files>nullrealm/tools/builtins/code_search.py, nullrealm/context/assembler.py, nullrealm/context/graph.py</files>
  <action>
  Build code_search as an agent tool:
  - nullrealm/tools/builtins/code_search.py:
    - Extends BaseTool, registered in tool registry
    - Input: {"query": "...", "repo": "...", "k": 10}
    - Output: list of code chunks with file path, line numbers, similarity score, snippet
    - Wraps PgVectorStore.search()

  Build context assembly pipeline:
  - nullrealm/context/assembler.py:
    - ContextAssembler class
    - assemble(task, repos) → assembled context string
    - Strategy:
      1. Always include: REPO_INDEX.md for all repos (~7k tokens)
      2. On-demand: LLM-selected per-file summaries based on task
      3. Fallback: pgvector semantic search for task-relevant code
      4. Live: agent can use file_read, grep, code_search tools during execution

  Build cross-repo dependency graph:
  - nullrealm/context/graph.py:
    - Extract from imports, proto files, infra-as-code
    - Store as JSONB in PostgreSQL (service→service edges with type: gRPC, HTTP, etc.)
    - query_dependencies(service) → list of upstream/downstream services
    - Add as agent tool: query_repo_graph
  </action>
  <verify>python -c "from nullrealm.tools.builtins.code_search import CodeSearchTool; t = CodeSearchTool(); print(t.execute({'query': 'health endpoint', 'k': 3}))"</verify>
  <done>code_search tool returns relevant code, context assembler produces rich context, graph queryable</done>
