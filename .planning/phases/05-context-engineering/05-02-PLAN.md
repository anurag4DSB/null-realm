  <name>Task 1: MCP server with Google OAuth (Streamable HTTP transport)</name>
  <files>nullrealm/mcp_server.py, nullrealm/mcp_auth.py, Dockerfile.mcp, infra/k8s/gke/mcp/</files>
  <action>
  Build the MCP server following the 2025-06-18 spec (Streamable HTTP transport):

  Architecture:
  ```
  Claude Code / Any MCP client
       │
       │  POST/GET/DELETE /mcp
       │  Authorization: Bearer <token>
       │
       ▼
  mcp.34.53.165.155.nip.io
       │
       ├─ Traefik (subdomain routing, NO OAuth2 Proxy middleware)
       │  (MCP server handles its own auth — token-based, not cookie-based)
       │
       ▼
  MCP Server (FastAPI)
       │
       ├─ /mcp              → MCP Streamable HTTP endpoint
       │   POST: client sends JSON-RPC messages
       │   GET:  client opens SSE stream
       │   DELETE: client terminates session
       │
       ├─ /oauth/authorize   → Redirect to Google OAuth consent
       ├─ /oauth/callback    → Google redirects here with auth code
       ├─ /oauth/token       → Exchange auth code for MCP session token
       │
       ├─ pgvector           → code_search tool
       ├─ Neo4j              → graph_query, graph_path, service_map tools
       └─ REPO_INDEX.md      → repo://null-realm/index resource
  ```

  MCP Server implementation (nullrealm/mcp_server.py):
  - Use `mcp` Python package (pip install mcp) for MCP protocol handling
  - OR build from scratch with FastAPI + SSE (the spec is simple JSON-RPC over HTTP)
  - Register 4 tools: code_search, graph_query, graph_path, service_map
  - Register 2 resources: repo://null-realm/index, repo://null-realm/graph
  - Session management: Mcp-Session-Id header, in-memory session store
  - Stdio mode: `python -m nullrealm.mcp_server --stdio` for local Claude Code

  OAuth implementation (nullrealm/mcp_auth.py):
  - Reuse SAME Google OAuth Client ID + Secret (from OAuth2 Proxy secrets)
  - /oauth/authorize: redirect to Google with client_id, redirect_uri, scope=email+profile
  - /oauth/callback: receive auth code from Google, exchange for Google access token
  - /oauth/token: issue a signed MCP session token (JWT with email, expiry)
  - Token verification: check JWT signature + expiry on every /mcp request
  - Redirect URI: http://mcp.34.53.165.155.nip.io/oauth/callback
    (user must add this to Google Console → Credentials → null-realm)

  How Claude Code authenticates:
  1. User adds MCP server URL in Claude Code
  2. Claude Code sends POST /mcp → server returns 401 + OAuth metadata
  3. Claude Code opens browser → Google login → user approves
  4. Google redirects to /oauth/callback with code
  5. MCP server exchanges code → issues JWT token
  6. Claude Code stores token, sends with all subsequent requests

  Dockerfile.mcp:
  - python:3.12-slim + uv
  - COPY nullrealm/ + repo-indexes/
  - CMD: uv run python -m nullrealm.mcp_server --port 8090
  - Expose port 8090

  K8s manifests (GKE):
  - infra/k8s/gke/mcp/deployment.yaml: image from Artifact Registry, env vars for DB/Neo4j/OAuth
  - infra/k8s/gke/mcp/service.yaml: ClusterIP on port 8090
  - Traefik IngressRoute: mcp.34.53.165.155.nip.io → mcp service port 8090
    NOTE: NO oauth2-auth middleware — MCP server handles its own auth (token-based)

  K8s secrets needed:
  - GOOGLE_OAUTH_CLIENT_ID: same as OAuth2 Proxy
  - GOOGLE_OAUTH_CLIENT_SECRET: same as OAuth2 Proxy
  - MCP_JWT_SECRET: random 256-bit key for signing session tokens

  Dependencies: mcp (or build with FastAPI), PyJWT
  </action>
  <verify>

  3. After OAuth flow: curl -H "Authorization: Bearer <token>" http://mcp.34.53.165.155.nip.io/mcp → 200

<task type="auto">
  <name>Task 2: MCP tools — code_search + graph_query + graph_path + service_map</name>
  <files>nullrealm/mcp_tools.py</files>
  <action>
  Register 4 MCP tools that wrap the existing pgvector + Neo4j stores:

  - code_search:
    - Input: {"query": str, "repo": str (optional), "k": int (default 10)}
    - Calls PgVectorStore.search()
    - Returns: list of {file_path, symbol_name, symbol_type, score, code_snippet, line_start, line_end}

  - graph_query:
    - Input: {"symbol": str, "depth": int (default 2)}
    - Calls Neo4jStore.query_neighbors()
    - Returns: list of {file, name, type, distance, relationship}

  - graph_path:
    - Input: {"source": str, "target": str}
    - Calls Neo4jStore.query_path()
    - Returns: {path_nodes: [...], edge_types: [...]}

  - service_map:
    - Input: {} (no params)
    - Calls Neo4jStore.query_service_map()
    - Returns: list of {source_file, relationship, target_file}

  Register 2 MCP resources:
  - repo://null-realm/index → contents of repo-indexes/null-realm/REPO_INDEX.md
  - repo://null-realm/graph → JSON export of full Neo4j graph (nodes + edges)

  Also create LangGraph tool wrappers (for our own agents):
  - nullrealm/tools/builtins/code_search.py — LangChain @tool wrapping same logic
  - nullrealm/tools/builtins/graph_query.py — LangChain @tool wrapping same logic
  - Register in agent_configs/tools/ seed data
  - Add to research assistant's tool_allowlist
