<task type="auto">
  <name>Task 2: Context assembler with hybrid retrieval</name>
  <files>nullrealm/context/assembler.py</files>
  <action>
  Build hybrid context assembly (vector + graph):

  - nullrealm/context/assembler.py:
    - ContextAssembler class
    - assemble(task, repos) → AssembledContext

    Strategy:
    1. Always include: REPO_INDEX.md for relevant repos
    2. Vector search: pgvector semantic search for task-relevant code chunks
    3. Graph expansion: for each vector hit, query Neo4j for connected symbols (imports, callers, dependents)
    4. Rank + deduplicate: merge vector results + graph-expanded results, rank by relevance
    5. Output: structured context with provenance (where each piece came from)

    - AssembledContext dataclass:
      - repo_summary: str (REPO_INDEX.md content)
      - vector_results: list[CodeChunk] (from pgvector, with scores)
      - graph_paths: list[GraphPath] (from Neo4j, with relationship types)
      - total_tokens: int (context budget tracking)

  Wire into agent bootstrap:
  - Before agent loop, run context assembler to pre-fetch relevant context
  - Include assembled context in the system prompt
  - Agent can still use code_search/graph_query tools for on-demand exploration
  </action>
  <verify>python -c "from nullrealm.context.assembler import ContextAssembler; a = ContextAssembler(); ctx = a.assemble('How does auth work?', ['null-realm']); print(len(ctx.vector_results), len(ctx.graph_paths))"</verify>
  <done>Context assembler produces hybrid results from both pgvector and Neo4j</done>
</task>
