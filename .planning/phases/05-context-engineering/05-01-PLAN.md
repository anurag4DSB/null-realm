  <name>Task 1: AST parsing + code chunking + Vertex AI embeddings + pgvector</name>
  <files>nullrealm/context/__init__.py, nullrealm/context/indexer.py, nullrealm/context/pgvector_store.py, nullrealm/context/embeddings.py</files>
  <action>
  Build the indexing pipeline:

  - nullrealm/context/embeddings.py:
    - Embed text using Vertex AI text-embedding-005 (768 dimensions)
    - User has admin access to Vertex AI on GCP project helpful-rope-230010
    - Batch embedding support (Vertex AI handles batches of up to 250 texts)
    - Fallback: sentence-transformers/all-MiniLM-L6-v2 for local dev (no Vertex AI needed)

  - nullrealm/context/indexer.py:
    - Language-aware AST parsing (Python ast module for .py files)
    - Split code into semantic chunks: functions, classes, modules (~200-500 tokens each)
    - Extract relationships: imports, function calls, class inheritance
    - Attach metadata: file path, language, function name, class name, imports list
    - Output: list of CodeChunk + list of CodeRelationship

  - nullrealm/context/pgvector_store.py:
    - PgVectorStore class wrapping SQLAlchemy + pgvector
    - Table: code_embeddings (id, chunk_text, embedding vector(768), repo, file_path, symbol_name, symbol_type, metadata JSONB)
    - store_embeddings(chunks) → batch insert with embeddings
    - search(query, k=10) → list of chunks with cosine similarity scores
    - HNSW index for fast ANN (<50ms on 500k vectors)

  CLI: python -m nullrealm.context.indexer --repo . --embed
  Start by indexing null-realm itself.

  Dependencies to add: google-cloud-aiplatform, pgvector (SQLAlchemy extension)
  </action>
  <verify>python -m nullrealm.context.indexer --repo . --embed && python -c "from nullrealm.context.pgvector_store import PgVectorStore; s = PgVectorStore(); print(len(s.search('FastAPI health endpoint')))"</verify>
  <done>null-realm repo indexed, pgvector search returns relevant code chunks with scores</done>
</task>

<task type="auto">
  <name>Task 5: Human-in-the-loop review interface</name>
  <files>nullrealm/context/review.py, ui/review_app.py</files>
  <action>
  Build a review workflow so humans can inspect and correct the index before agents use it.

  The pipeline is iterative, NOT one-shot:
  Index → Visualize → Human reviews → Correct → Re-index → Better results

  Review interface (Streamlit tab or standalone page):
  - Graph review:
    - Show Neo4j edges in a table: source → relationship → target
    - Human can approve, reject, or add edges
    - Rejected edges get removed from Neo4j on save
    - Missing edges can be added manually (e.g., "service-a CALLS service-b via HTTP")

  - Chunk review:
    - Show code chunks with their embeddings' nearest neighbors
    - Human can mark chunks as: keep, noise (excluded from search), or split (re-chunk)
    - Spotlight integration: outlier chunks flagged automatically for review

  - Summary review:
    - Show REPO_INDEX.md in editable markdown
    - Human edits inline, saves back to disk
    - Trigger re-generation if needed

  - Re-index button:
    - After corrections, re-run: embeddings for modified chunks, graph updates, PaCMAP re-projection
    - Incremental: only re-process what changed

  Deploy as a tab in the embeddings Streamlit app:
  - embeddings.34.53.165.155.nip.io/review → review interface

  Future (not this phase): PR webhook that auto-triggers re-indexing on code changes.
  This is documented in docs/open-questions.md for later implementation.
  </action>
  <verify>Open review interface, approve/reject a few graph edges, mark a chunk as noise, verify changes persist</verify>
  <done>Human can review and correct the index via a web UI, corrections feed back into the pipeline</done>
