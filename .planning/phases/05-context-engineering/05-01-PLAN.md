<task type="auto">
  <name>Task 1: AST parsing + code chunking + embedding</name>
  <files>nullrealm/context/indexer.py, nullrealm/context/pgvector_store.py</files>
  <action>
  Build the indexing pipeline:
  - nullrealm/context/indexer.py:
    - Language-aware AST parsing (Python ast module for .py, tree-sitter for JS/TS/Go)
    - Split code into semantic chunks: functions, classes, modules (~200-500 tokens each)
    - Attach metadata: file path, language, function name, class, imports
    - Embed chunks using text-embedding-005 via Vertex AI (768 dimensions)
    - Store: (chunk_text, embedding_vector, metadata) in pgvector

  - nullrealm/context/pgvector_store.py:
    - PgVectorStore class wrapping SQLAlchemy + pgvector
    - store_embeddings(chunks: list[CodeChunk]) → batch insert
    - search(query: str, k: int = 10) → list[CodeChunk] with similarity scores
    - Use HNSW index for fast approximate nearest neighbor (<50ms on 500k vectors)

  Start by indexing null-realm itself. Add other repos via CLI command.
  Use batch processing for embeddings (Vertex AI supports batches).
  </action>
  <verify>python -m nullrealm.context.indexer --repo . && python -c "from nullrealm.context.pgvector_store import PgVectorStore; s = PgVectorStore(); print(len(s.search('FastAPI health endpoint')))"</verify>
  <done>null-realm repo indexed, pgvector search returns relevant code chunks</done>
</task>

    - Also store locally for local dev

  Create CLI command: python -m nullrealm.context.summaries --repo . --output repo-indexes/
