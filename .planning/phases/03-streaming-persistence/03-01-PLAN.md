<task type="auto">
  <name>Task 1: NATS JetStream deployment + messaging client</name>
  <files>infra/k8s/system/nats/statefulset.yaml, infra/k8s/system/nats/service.yaml, nullrealm/communication/nats_bus.py, nullrealm/communication/events.py</files>
  <action>
  Deploy NATS with JetStream on Kind:
  - StatefulSet: nats:latest with JetStream enabled (-js flag), 3 replicas for durability
  - Service: ClusterIP on ports 4222 (client), 8222 (monitoring)
  - PVCs: 1Gi per replica for JetStream storage

  Create messaging client:
  - nullrealm/communication/events.py: Pydantic event schemas
    - TextDeltaEvent: {"type": "text_delta", "content": "...", "session_id": "..."}
    - ToolUseEvent: {"type": "tool_use", "tool": "...", "input": {...}, "session_id": "..."}
    - ToolResultEvent: {"type": "tool_result", "tool": "...", "output": "...", "session_id": "..."}
    - TaskCompleteEvent: {"type": "task_complete", "result": "...", "session_id": "..."}
  - nullrealm/communication/nats_bus.py:
    - NATSBus class with connect(), publish(subject, event), subscribe(subject, callback)
    - Subjects: agent.{session_id}.events, ctrl.{session_id}, done.{session_id}
    - Use JetStream for durable delivery (messages survive pod restarts)
  </action>
  <verify>kubectl get pods -n null-realm | grep nats && python -c "import asyncio; from nullrealm.communication.nats_bus import NATSBus; print('ok')"</verify>
  <done>NATS running on Kind, NATSBus client connects and can pub/sub</done>
</task>

<task type="auto">
  <name>Task 2: Agent streaming via NATS</name>
  <files>nullrealm/worker/langgraph_agent.py, nullrealm/api/websocket.py</files>
  <action>
  Rewire the agent-to-UI flow through NATS:
  - Agent side (langgraph_agent.py):
    - On each LLM token: publish TextDeltaEvent to agent.{session_id}.events
    - On tool use: publish ToolUseEvent
    - On tool result: publish ToolResultEvent
    - On completion: publish TaskCompleteEvent to done.{session_id}
  - API server side (websocket.py):
    - On WebSocket connect: subscribe to agent.{session_id}.events
    - Forward all events to WebSocket client as they arrive
    - Handle ctrl.{session_id} for interrupts (user sends cancel)
  - Chainlit side (ui/app.py):
    - Handle text_delta events for streaming display
    - Handle tool_use/tool_result for step visualization

  For now, agent still runs in-process in API server — NATS is the message bus between components.
  Pod-based agent execution comes in Phase 04 with Argo.
  </action>
  <verify>Send chat message, check NATS monitoring: curl localhost:8222/connz | python -m json.tool</verify>
  <done>Tokens stream character-by-character in Chainlit, tool steps shown inline, NATS subjects active</done>
</task>
